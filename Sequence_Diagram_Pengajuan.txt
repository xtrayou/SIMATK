Title: Sequence Diagram - Pengajuan Permintaan ATK

# Aktor dan Komponen
actor Pegawai
participant View (Form Publik)
participant PermintaanController
participant BarangModel
participant PermintaanModel
database Database

# Alur (Sequence)
Pegawai->View (Form Publik): Mengisi Form & Menekan Submit
View (Form Publik)->PermintaanController: HTTP POST (askStore)
PermintaanController->BarangModel: Cek Ketersediaan Stok (getBarang)
BarangModel->Database: Query Cek Stok
Database-->BarangModel: Return Data Stok
BarangModel-->PermintaanController: Return Status Stok (Tersedia/Tidak)

# Kondisi (Alt/Opt)
opt Jika Stok Tersedia
    PermintaanController->PermintaanModel: Buat Kode Resi
    PermintaanController->PermintaanModel: Insert Permintaan Utama
    PermintaanModel->Database: INSERT ke tabel requests
    Database-->PermintaanModel: Return Insert ID
    PermintaanController->PermintaanModel: Insert Detail Barang
    PermintaanModel->Database: INSERT ke tabel request_items
    Database-->PermintaanModel: Success Response
    PermintaanModel-->PermintaanController: Return Sukses
    PermintaanController-->View (Form Publik): Redirect ke Halaman Sukses
    View (Form Publik)-->Pegawai: Tampilkan Pesan Sukses & Kode Resi
end

opt Jika Stok Tidak Tersedia
    PermintaanController-->View (Form Publik): Redirect Error
    View (Form Publik)-->Pegawai: Tampilkan Pesan "Stok Habis"
end
